## 📦 白嫖存储（6种渠道）

### 1. Telegram（推荐指数 ⭐⭐⭐⭐⭐）

**原理**：利用 Telegram Bot 将图片发送到频道，获得永久 `file_id`，通过代理转换为可访问链接。  
**优点**：无限空间，单文件最大 50MB，稳定不删图。  
**缺点**：原始直链被墙，需配合代理（见后文）。

**获取步骤**：
1. **创建 Bot**：打开 Telegram，搜索 `@BotFather`，发送 `/newbot`，按提示输入机器人名称和用户名（必须以 `bot` 结尾）。成功后你会收到一条消息，其中包含 **Bot Token**（如 `123456:ABCdefg`）。**复制并保存**。
2. **创建频道**：在 Telegram 中创建一个新的**私有频道**（任意名称），记下频道名。
3. **邀请机器人**：打开频道信息页 → 成员 → 添加成员，搜索你刚创建的机器人用户名，将其添加为**管理员**（至少需要“发送消息”和“上传文件”权限）。
4. **获取频道 ID**：在频道中随便发一条消息（例如“test”）。然后在浏览器地址栏打开以下链接（将 `你的BotToken` 替换为真实 Token）：
   ```
   https://api.telegram.org/bot你的BotToken/getUpdates
   ```
   页面会显示一段 JSON 数据，找到 `"chat":{"id":-1001234567890,...` 其中的 **`id` 值**（是一个负数，例如 `-1001234567890`）。复制该数字。
5. **填入 ImgBed**：在 ImgBed 后台「存储管理」中选择 Telegram，填入 Bot Token 和 Channel ID，保存。

**注意**：
- 频道必须为私有，否则图片可能被他人看到。
- 机器人必须具有发送消息和上传文件的权限。
- 如需删除图片，直接在 Telegram 频道中删除对应消息即可。

**官方参考**：[Telegram Bot API 文档](https://core.telegram.org/bots)

### 2. Discord（推荐指数 ⭐⭐⭐⭐）

**原理**：Discord 将上传的文件托管在 `cdn.discordapp.com` 全球 CDN 上，通过 Bot 发送消息到频道即可获得链接。  
**优点**：无限存储，单文件最大 25MB，自带 CDN。  
**缺点**：国内部分地区可能无法直接访问 CDN 域名，需配合代理。

**获取步骤**：
1. **创建应用**：访问 [Discord Developer Portal](https://discord.com/developers/applications)，点击「New Application」，输入名称（任意）。
2. **创建 Bot**：左侧菜单选择「Bot」，点击「Add Bot」→「Reset Token」并**复制 Bot Token**（类似 `MTAzNzI...`）。
3. **设置 Bot 权限**：在 Bot 页面下方找到「Privileged Gateway Intents」，开启 **「Message Content Intent」**（用于读取消息内容，可选但建议开启）。
4. **邀请 Bot 到服务器**：左侧菜单选择「OAuth2」→「URL Generator」，勾选 `bot` 和 `Send Messages`、`Attach Files`、`Read Messages`、`Manage Messages` 权限，复制下方生成的邀请链接，在浏览器中打开，选择你的服务器（需拥有管理员权限），完成邀请。
5. **获取频道 ID**：在 Discord 客户端中打开「用户设置」→「高级」→ 开启「开发者模式」。然后右键点击你准备用于存储图片的**文字频道**，选择「复制频道 ID」，获得一串数字（例如 `123456789012345678`）。
6. **填入 ImgBed**：在 ImgBed 后台添加 Discord 存储，填入 Bot Token 和 Channel ID。

**注意**：
- Bot 必须拥有发送消息和上传文件的权限。
- 建议为图片单独创建一个频道，避免与其他聊天内容混在一起。
- Discord 的 CDN 链接有效期很长（理论永久），但若消息被删除则链接失效。

**官方参考**：[Discord Developer Portal](https://discord.com/developers/docs/intro)

### 3. Hugging Face（推荐指数 ⭐⭐⭐）

**原理**：Hugging Face 的 Model 仓库支持 Git LFS，可以存储任意文件，并通过 `raw` 链接公开访问。  
**优点**：1GB 免费存储（每个仓库），技术社区常用。  
**缺点**：存储空间较小，国内访问速度一般，需配合代理。

**获取步骤**：
1. **注册账号**：访问 [Hugging Face](https://huggingface.co/) 注册一个免费账号。
2. **创建 Model 仓库**：点击右上角头像 → 「New Model」。
   - 输入模型名称（如 `my-imgbed`）。
   - **Visibility** 必须选择 **「Public」**。
   - 其他默认，点击「Create model」。
3. **生成 Access Token**：点击右上角头像 → 「Settings」→ 「Access Tokens」→ 「New token」。
   - 名称任意，权限勾选 **「write」**（至少需要 write 才能上传文件）。
   - 点击「Generate token」，**复制并保存**（形如 `hf_xxxxxx`）。
4. **获取仓库名称**：你的仓库名称格式为 `用户名/模型名`（例如 `stevenrao/my-imgbed`）。
5. **填入 ImgBed**：在 ImgBed 后台添加 Hugging Face 存储，填入 Access Token 和仓库名称。

**注意**：
- 每个模型仓库有 1GB 免费 LFS 存储，可以创建多个仓库。
- 上传的文件会出现在仓库的「Files」选项卡中，可手动删除。
- 原始直链地址为 `https://huggingface.co/用户名/模型名/resolve/main/文件名`，需配合代理使用。

**官方参考**：[Hugging Face 文档](https://huggingface.co/docs/hub/en/models-uploading)

### 4. Cloudflare R2（推荐指数 ⭐⭐⭐⭐）

**原理**：Cloudflare R2 提供 S3 兼容的对象存储，免费额度足够个人图床。  
**优点**：10GB 免费存储 + 每月 10GB 出站流量，全球 CDN 加速。  
**缺点**：需要绑定信用卡（但不会扣费，仅验证身份），免费额度用尽后按量计费（可设置限额）。

**获取步骤**：
1. **注册 Cloudflare**：访问 [Cloudflare](https://cloudflare.com) 注册账号。
2. **开通 R2**：登录后，左侧菜单选择「R2」→ 点击「开始使用」→ 根据提示添加付款方式（支持信用卡，不会扣费，仅用于防滥用）。
3. **创建存储桶**：点击「创建存储桶」，输入名称（如 `imgbed`），区域选择「自动」，点击创建。
4. **生成 API 令牌**：进入存储桶详情页，点击「设置」→「API 令牌」→「创建 API 令牌」。
   - 权限选择「对象读写」（Object Read & Write）。
   - 令牌名称任意。
   - 创建后会显示 **Access Key ID** 和 **Secret Access Key**，**务必立即复制保存**（关闭后无法再查看 Secret）。
5. **获取端点**：R2 的默认端点为 `https://<你的账户ID>.r2.cloudflarestorage.com`。账户ID可以在 R2 概览页或 Workers 仪表板找到。
6. **填入 ImgBed**：在 ImgBed 后台选择「S3 兼容存储」，填入：
   - 端点（Endpoint）：`https://你的账户ID.r2.cloudflarestorage.com`
   - 区域（Region）：`auto`
   - 存储桶名称：你创建的桶名
   - 访问密钥（Access Key ID）
   - 秘密密钥（Secret Access Key）

**注意**：
- 免费额度每月重置，超出部分会收费，建议在 Cloudflare 设置中启用「限制 R2 使用量」。
- 通过 ImgBed 代理访问时，流量走代理，不计入 R2 出站流量，可节省额度。

**官方参考**：[Cloudflare R2 文档](https://developers.cloudflare.com/r2/)

### 5. S3 兼容存储（其他免费选项）

以下存储均支持标准 S3 协议，ImgBed 可直接对接。部分有永久免费额度，部分为限时试用。

| 服务商 | 免费额度 | 是否需要信用卡 | 配置要点 |

| **Backblaze B2** | 10GB 存储 + 1GB/天 下行流量 | 是（验证身份） | 需创建存储桶，生成应用密钥，端点：`https://s3.us-west-002.backblazeb2.com` |
| **腾讯云 COS** | 50GB 存储（6个月） | 是（实名） | 需开通 COS 服务，创建存储桶，获取 SecretId/SecretKey |

4. 在 ImgBed 的 S3 兼容配置中填入对应的端点、区域（如 `us-west-002`）、存储桶名称、密钥对。

### 6. 网盘挂载（Alist + 阿里云盘 / 夸克网盘等）

**原理**：使用 [Alist](https://alist.nn.ci/) 将网盘挂载为 WebDAV 或 S3 接口，ImgBed 通过 WebDAV 连接，实现超大容量存储。  
**优点**：网盘容量大（通常 2TB+），单文件可达 100MB+。  
**缺点**：需要额外维护 Alist 服务（可运行在本地或免费云）；部分网盘直链有时效性，但 Alist 可自动刷新。

**获取步骤**：
1. **部署 Alist**：
   - **桌面运行**：下载 Alist 二进制文件，双击运行（Windows）或终端执行（Mac/Linux）。
   - **Docker 运行**：`docker run -d --restart=always -p 5244:5244 -v /etc/alist:/opt/alist/data xhofe/alist:latest`
   - **免费云托管**：可使用 Railway 或 Render 免费部署 Alist（需绑定 GitHub）。
2. **添加网盘存储**：打开 Alist 管理后台（默认 `http://localhost:5244`，初始密码可在日志中查看），进入「存储」→「添加」。
   - 选择驱动类型（如「阿里云盘」），根据提示获取 Refresh Token（通常通过扫码登录）。
   - 设置挂载路径（如 `/ali`）。
3. **开启 WebDAV**：Alist 默认开启 WebDAV 服务，地址为 `http://localhost:5244/dav`。用户名和密码与你登录 Alist 后台的相同。
4. **填入 ImgBed**：在 ImgBed 后台选择「WebDAV」，填入：
   - 端点：`http://你的Alist地址/dav`（如 `http://localhost:5244/dav`）
   - 用户名：Alist 用户名（默认 `admin`）
   - 密码：你的 Alist 密码
   - 路径前缀：可留空或填 `/ali`（根据你挂载的路径）

**注意**：
- 如果 Alist 部署在本地，ImgBed 需能访问该地址（同一台机器可直接用 `localhost`）。
- 网盘直链有效期由网盘决定，Alist 会在访问时自动刷新，因此 ImgBed 获取的链接会实时有效。
- 建议选择支持 Alist 官方文档中标记为「稳定性高」的网盘（如阿里云盘、夸克网盘）。

**官方参考**：
- [Alist 文档](https://alist.nn.ci/zh/)
- [阿里云盘 Refresh Token 获取教程](https://alist.nn.ci/zh/guide/drivers/aliyundrive.html)

## ⚡ 白嫖代理（一键部署，无需代码）

我们已为你写好了代理程序，存储在 GitHub 仓库 [imgbed-proxy](https://github.com/mageg-x/imgbed-proxy)。你只需点一下按钮，即可将代理部署到免费平台，获得专属代理域名。

### 方案一：Cloudflare Workers（推荐）

**特点**：全球边缘网络，每日 10 万次请求免费，绑定自定义域名后国内访问速度尚可。

**步骤**：
1. 点击下方按钮，登录你的 Cloudflare 账号（如果没有，免费注册一个）。
   [![Deploy to Cloudflare Workers](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/mageg-x/imgbed-proxy)
2. 按页面提示为 Worker 命名（例如 `imgbed-proxy`），点击「部署」。
3. 部署成功后，你会获得一个形如 `https://你的项目名.workers.dev` 的域名。**复制该域名**。
4. 回到 ImgBed 后台的「代理管理」，粘贴该地址，保存。

### 方案二：Vercel Edge Functions（备选）

**特点**：每月 100GB 免费流量，但默认域名在国内可能被 DNS 污染，建议搭配 Cloudflare CDN。

**步骤**：
1. 点击下方按钮，登录 Vercel（可使用 GitHub 账号快速登录）。
   [![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/mageg-x/imgbed-proxy)
2. 点击「Deploy」，等待部署完成（约 1-2 分钟）。
3. 部署后，Vercel 会分配一个 `https://你的项目名.vercel.app` 域名。**复制该域名**。
4. 在 ImgBed 后台的「代理管理」中填入该地址。

**提示**：
- 两个平台均可长期免费使用，个人图床完全够用。
- 如果担心 Vercel 域名被墙，可将 Cloudflare Workers 作为首选。

## 🌐 白嫖 CDN（可选，进一步提升国内访问速度）

> 注意：大多数情况下，Cloudflare Workers 自带的 CDN 已经够用。如果你对国内访问速度有更高要求，可以叠加以下免费 CDN。

### 1. 使用 Cloudflare 自带 CDN（最简单）

你部署的 Workers 域名本身就在 Cloudflare 网络上，国内访问走的是 Cloudflare 的跨境链路。如果想用自己的域名，可以：
- 将你的自定义域名（需备案或境外）通过 CNAME 指向 `你的项目名.workers.dev`，并在 Cloudflare DNS 管理中开启「代理」（橙色云朵）。这样国内用户访问你的域名时会经过 Cloudflare 节点。

### 2. 又拍云 / 七牛云 免费 CDN 额度

| 服务商 | 免费额度 | 条件 | 配置方式 |
|--------|----------|------|----------|
| **又拍云** | 每月 15GB CDN 流量 + 15GB 存储 | 需在网站底部添加又拍云 Logo | 将你的代理域名作为源站，添加 CDN 服务，获得加速域名 |
| **七牛云** | 每月 10GB CDN 流量 + 10GB 存储 | 需完成实名认证 | 类似又拍云，创建 CDN 加速域名，源站填代理地址 |

**操作步骤**（以又拍云为例）：
1. 注册又拍云账号，完成实名认证。
2. 进入「CDN」服务，点击「添加域名」。
3. 加速域名填写你希望使用的域名（需备案或境外）。
4. 源站地址填写你的代理域名（如 `xxx.workers.dev`），协议选择 HTTPS。
5. 提交后，根据提示将你的加速域名 CNAME 到又拍云提供的地址。
6. 在 ImgBed 代理配置中，将代理地址改为又拍云分配的加速域名（注意：ImgBed 需要完整的 `https://加速域名` 格式）。

**注意**：又拍云需要挂载标识，七牛云需要实名。免费额度每月重置，超量会暂停或扣费，建议设置告警。

### 3. 腾讯云 EdgeOne 免费试用（短期）

- 提供 10GB 流量 + 10万次请求，有效期 14 天（可重复申请？）。
- 需域名备案，适合短期测试。

**结论**：对于绝大多数用户，直接使用 Cloudflare Workers 代理即可，无需额外 CDN。

## ⚙️ 最后：配置到 ImgBed

1. **下载并运行 ImgBed**（桌面版或服务端）：
   - 桌面版：下载对应操作系统的安装包，双击运行，系统托盘出现图标。
   - 服务端：下载二进制文件，终端执行 `./imgbed`。
2. **打开管理后台**：浏览器访问 `http://localhost:8380`，首次使用需要设置管理员密码。
3. **添加存储渠道**：
   - 进入「存储管理」，点击「添加存储」。
   - 选择你白嫖到的渠道（Telegram/Discord/R2 等），填入对应的 Token/密钥。
   - 保存后，可以点击「测试」验证连接是否正常。
4. **配置代理**：
   - 进入「代理管理」，点击「添加代理」。
   - 填入你部署好的代理域名（例如 `https://imgbed-proxy.workers.dev`）。
   - 勾选「启用代理转换」，保存。
5. **上传测试**：
   - 进入「上传」页面，选择一张图片上传。
   - 上传成功后，ImgBed 会显示原始直链和转换后的代理链接。
   - 复制代理链接，在国内浏览器中打开，确认图片正常显示。

## ❓ 常见问题

**Q：我部署的 Cloudflare Worker 代理访问时返回 404？**  
A：检查 ImgBed 中填写的代理地址是否正确（不要遗漏 `https://`），并确认代理已启用。另外，ImgBed 生成的链接格式为 `https://代理域名/base58(原始host)/图片路径`，如果手动修改过代理代码导致路由变化，请重新部署。

**Q：Telegram 频道 ID 是负数，ImgBed 支持吗？**  
A：支持，直接粘贴即可。

**Q：Discord 的 Bot Token 重置后，之前的图片还能访问吗？**  
A：可以，只要频道和消息还在，图片链接依然有效。但新上传需要使用新 Token。

**Q：Hugging Face 上传后，直接访问原始链接被限制？**  
A：需要登录才能访问？请确认你的仓库是 Public 状态。另外 Hugging Face 有时会要求登录才能下载，建议始终通过代理访问。

**Q：网盘挂载方式会不会很慢？**
